# EnergyPac ERP - Complete Purchase Flow API Documentation

Base URL: `/api/`  
Authentication: JWT Bearer Token (8-hour expiry)  
Header: `Authorization: Bearer <access_token>`

---

## Step 1: Create Requisition (Purchase Request)

A requisition is the starting point. It contains items that need to be purchased.  
When items are added to a requisition, they are automatically linked in the Item Master (`requisition_number` is saved on the product).

**Create Requisition**
```
POST /api/requisitions
```
```json
{
  "requisition_number": "EEL/2026/001",
  "requisition_date": "2026-05-22",
  "remarks": "Urgent requirement for project X",
  "items": [
    {
      "product": "product-uuid",
      "quantity": 100,
      "remarks": "Need high quality"
    }
  ]
}
```
- `requisition_number` is optional — auto-generates as `EEL/YEAR/NUMBER` if not provided
- If provided, it must be unique
- Each product used in the requisition gets `requisition_number` saved in the Item Master automatically

**List Requisitions**
```
GET /api/requisitions
```
Query params: `?is_assigned=true&requisition_date=2026-05-22&search=EEL&ordering=-requisition_date`

**Get Requisition Detail**
```
GET /api/requisitions/{id}
```

**Get Requisition Items**
```
GET /api/requisitions/{id}/items
```

## Item Master (Products)

Every item in the system is stored in the Item Master. Each item tracks which requisition it belongs to.

**Create Item in Item Master**
```
POST /api/products
```
```json
{
  "item_name": "Steel Rod 10mm",
  "description": "High tensile steel rod",
  "hsn_code": "7214",
  "unit": "KG",
  "rate": 250.00,
  "requisition_number": "EEL/2026/001"
}
```
- `item_code` is auto-generated (e.g., `ITEM/0001`)
- `requisition_number` links this item to a requisition

**List All Items**
```
GET /api/products
```
Query params: `?is_active=true&unit=KG&requisition_number=EEL/2026/001&search=steel`

**Get Items by Requisition Number**
```
GET /api/products/by_requisition?requisition_number=EEL/2026/001
```
Returns all items linked to a specific requisition.

**Get Single Item**
```
GET /api/products/{id}
```

**Update Item**
```
PATCH /api/products/{id}
```

**Low Stock Items**
```
GET /api/products/low_stock
```

Assign one or more vendors to a requisition so they can quote prices.

**Create Vendor Assignment**
```
POST /api/vendor-assignments
```
```json
{
  "requisition": "requisition-uuid",
  "vendor": "vendor-uuid",
  "remarks": "Request quotation for all items",
  "items": [
    {
      "requisition_item": "requisition-item-uuid",
      "quantity": 100
    }
  ]
}
```
- One requisition can have multiple vendor assignments
- Each vendor can only be assigned once per requisition (unique constraint)

**List Assignments for a Requisition**
```
GET /api/requisitions/{id}/assignments
```

**Get Items Available for Quotation Entry**
```
GET /api/vendor-assignments/{id}/items_for_quotation
```
Returns the items the vendor needs to quote on.

## Step 3: Enter Vendor Quotation

After vendor submits their pricing, enter their quotation.

**Create Vendor Quotation**
```
POST /api/vendor-quotations
```
```json
{
  "requisition": "requisition-uuid",
  "vendor": "vendor-uuid",
  "currency": "INR",
  "reference_number": "VENDOR-REF-123",
  "validity_date": "2026-06-22",
  "payment_terms": "30 days",
  "delivery_terms": "Ex-works",
  "remarks": "Prices valid for 30 days",

    {
      "vendor_item": "vendor-requisition-item-uuid",
      "quoted_rate": 250.50
    }
  ]
}
```
- `currency` can be "INR" or "USD" (each vendor quotes in their preferred currency)
- `amount` is auto-calculated: quantity x quoted_rate
- `total_amount` is auto-calculated: sum of all item amounts

**List Quotations**
```
GET /api/vendor-quotations

```

**Filter by Vendor**

```

```

**Filter by Requisition + Vendor**
```
GET /api/vendor-quotations/by_requisition_vendor?requisition=uuid&vendor=uuid
```

## Step 6: Generate Purchase Order

Generate PO(s) from the comparison. Select specific quotation items — system creates separate PO per vendor automatically.

**Generate PO from Comparison**
```
POST /api/purchase-orders/generate_from_comparison
```
```json
{
  "requisition": "requisition-uuid",
  "po_date": "2026-05-22",
  "subject": "Steel materials for Phase 2",
  "project_name": "Kolkata Metro Phase 2",
  "bill_to": "EnergyPac Engineering Ltd, 12 Park Street, Kolkata 700016",
  "ship_to": "Site Office, Kolkata Metro Phase 2, Salt Lake, Kolkata 700091",
  "selections": [
    "quotation-item-uuid-1",
    "quotation-item-uuid-2",
    "quotation-item-uuid-3"
  ],
  "discount_amount": 20.00,
  "cgst_percentage": 9.00,
  "sgst_percentage": 9.00,
  "igst_percentage": 0.00,
  "conversion_rate": 83.50
}
```
- `subject` = optional — PO subject line
- `project_name` = optional — project this PO belongs to
- `bill_to` = optional — billing address
- `ship_to` = optional — shipping/delivery address
- `selections` = array of VendorQuotationItem UUIDs (pick items from any vendor)
- System groups items by vendor and creates one PO per vendor
- PO Number format: `EEL/IND/<VENDOR_PREFIX>/<NUMBER>` (e.g., `EEL/IND/ALOKA/100`)
- `discount_amount` = vendor discount in PO currency (default 0). Subtracted from total: `total = items_total + GST - discount`
- GST is calculated on items_total
- Currency is inherited from the vendor's quotation
- `conversion_rate` = current INR rate at PO creation time (e.g., 1 USD = 83.50 INR). Stored for record only, no conversion applied. For INR POs, leave null or skip.

**Response:**
```json
{
  "message": "2 Purchase Order(s) created",
  "purchase_orders": [
    {
      "id": "uuid",
      "po_number": "EEL/IND/ALOKA/100",
      "vendor_name": "Aloka Industries",
      "currency": "USD",
      "conversion_rate": 83.50,
      "items_total": 300.00,
      "discount_amount": 20.00,
      "cgst_amount": 27.00,
      "sgst_amount": 27.00,
      "igst_amount": 0.00,
      "total_amount": 334.00,
      "status": "PENDING"
    }
  ]
}
```

## Step 8: Edit PO (Full Edit with Lock & Revision Tracking)

PO is fully editable — items, quantities, rates, GST, remarks, date. Everything can be changed.

### 8a. Acquire Lock Before Editing
```
POST /api/purchase-orders/{id}/lock
```
- Lock expires after 30 minutes
- Returns 409 if another user holds the lock

### 8b. Edit PO (Full Edit)
```
PATCH /api/purchase-orders/{id}
```
```json
{
  "po_date": "2026-05-25",
  "remarks": "Updated delivery terms",
  "discount_amount": 500.00,
  "cgst_percentage": 12.00,
  "sgst_percentage": 12.00,
  "igst_percentage": 0.00,
  "items": [
    {
      "id": "existing-item-uuid",
      "product": "product-uuid",
      "quantity": 150,
      "rate": 275.00
    },
    {
      "product": "new-product-uuid",
      "quantity": 50,
      "rate": 300.00
    }
  ]
}
```

**What can be edited:**
- `po_date` — PO date
- `subject` — PO subject line
- `project_name` — project name
- `bill_to` — billing address
- `ship_to` — shipping address
- `remarks` — any notes
- `discount_amount` — vendor discount (in PO currency)
- `cgst_percentage`, `sgst_percentage`, `igst_percentage` — GST rates
- `items` — full item list (add new, modify existing, remove items)

**Item edit rules:**
- Send `id` to update an existing item (change product, quantity, rate)
- Omit `id` to add a new item
- Items not included in the array are deleted
- Items already marked as "received" cannot be edited or removed

**Auto behavior on edit:**
- On first edit: PO number gets "R" suffix (e.g., `EEL/IND/ALOKA/100R`)
- `revision_number` increments on each edit
- `items_total`, GST amounts, `total_amount` (items + GST - discount) all recalculated automatically
- Audit log entry created with old vs new values
- Must hold the lock to edit

### 8c. Release Lock After Editing
```
POST /api/purchase-orders/{id}/unlock
```

### 12d. What Gets Logged Automatically

| Module | Action | When |
|--------|--------|------|
| PurchaseOrder | CREATE | PO generated from comparison |
| PurchaseOrder | UPDATE | PO edited (items/rates/GST/remarks changed) |
| PurchaseOrder | UPDATE (CANCEL) | PO cancelled (includes stock reversal details) |
| TransportEntry | CREATE | New transport entry created |
| TransportEntry | UPDATE | Transport entry edited (cost items, status, etc.) |
| TransportEntry | UPDATE (MARK_DELIVERED) | Shipment marked as delivered |

---

### 12e. Audit Log Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `id` | UUID | Unique log entry ID |
| `user` | UUID | User who performed the action |
| `user_name` | string | Full name of the user |
| `action` | string | `CREATE` / `UPDATE` / `DELETE` |
| `model_name` | string | Module name (`PurchaseOrder`, `TransportEntry`, etc.) |
| `object_id` | string | UUID of the affected object |
| `object_repr` | string | Human-readable name (e.g., PO number + vendor) |
| `changes` | JSON | Old/new values or action-specific details |
| `timestamp` | datetime | When the action happened (ISO 8601) |

## Transport Module (Landed Cost Tracking)

Track all transport/logistics costs per PO — freight, loading, unloading, insurance, customs, etc. Calculates landed cost per item for profit/loss analysis.

### Create Transport Entry
```
POST /api/transport
```
```json
{
  "purchase_order": "po-uuid",
  "transporter_name": "Blue Dart Logistics",
  "transporter_contact": "9876543210",
  "vehicle_number": "MH12AB1234",
  "driver_name": "Raju",
  "driver_contact": "9876543211",
  "dispatch_date": "2026-05-22",
  "expected_delivery_date": "2026-05-25",
  "dispatch_from": "Mumbai",
  "dispatch_to": "Pune",
  "remarks": "Handle with care",
  "cost_items": [
    {
      "cost_type": "FREIGHT",
      "description": "Mumbai to Pune transport",
      "amount": 3500.00
    },
    {
      "cost_type": "LOADING",
      "amount": 500.00
    },
    {
      "cost_type": "UNLOADING",
      "amount": 500.00
    },
    {
      "cost_type": "INSURANCE",
      "description": "Transit insurance",
      "amount": 800.00
    }
  ]
}
```
- Transport number auto-generated: `TRN/YEAR/NUMBER`
- `total_cost` auto-calculated from cost items
- One PO can have multiple transport entries (multiple shipments)
- Cost types: `FREIGHT`, `LOADING`, `UNLOADING`, `INSURANCE`, `CUSTOMS`, `OCTROI`, `HANDLING`, `PACKAGING`, `TOLL`, `OTHER`

### List Transport Entries
```
GET /api/transport
```
Query params: `?purchase_order=uuid&status=IN_TRANSIT&search=TRN&ordering=-dispatch_date`

### Get Transport by PO
```
GET /api/transport/by_po?purchase_order=po-uuid
```

### Update Transport Entry (Full Edit)
```
PATCH /api/transport/{id}
```
```json
{
  "status": "IN_TRANSIT",
  "vehicle_number": "MH12CD5678",
  "cost_items": [
    {
      "id": "existing-cost-item-uuid",
      "cost_type": "FREIGHT",
      "amount": 4000.00
    },
    {
      "cost_type": "TOLL",
      "amount": 200.00
    }
  ]
}
```
- Send `id` to update existing cost item, omit to add new
- Items not in array are deleted
- `total_cost` recalculated automatically
- Audit log created on every update

### Mark as Delivered
```
POST /api/transport/{id}/mark_delivered
```
Sets status to DELIVERED and records actual delivery date.

## Key Business Rules

| Rule | Description |
|------|-------------|
| Currency | Managed via Currency Master. Each vendor quotes in their own currency. No conversion. |
| Comparison | All vendors shown together with their currency. Cross-currency visible. |
| Item Master | Every item tracks its `requisition_number`. Items auto-linked when requisition is created. |
| PO Number | Format: `EEL/IND/<VENDOR_FIRST_WORD>/<NUMBER>` starting from 100 |
| PO Discount | Vendor discount (in PO currency) subtracted from total. Default 0. |
| PO Edit | Fully editable — items, rates, quantities, GST, discount, date, remarks. Received items protected. |
| PO Revision | First edit appends "R" to PO number. revision_number increments each edit. |
| Edit Lock | 30-minute timeout. Must lock before editing. |
| Item Code | Auto-generated, must be unique across the system |
| Deletion | Requisitions, Assignments, Quotations, POs cannot be deleted (audit trail) |
| Password | Cancel PO and Record Payment require `confirm_password` in body |
| GST | CGST + SGST (intra-state) OR IGST (inter-state). Applied on items_total. Total = items_total + GST - discount. |
| Stock | Auto-updated when PO item marked as purchased. Reversed on PO cancel. |
| Transport | Multiple shipments per PO. Cost allocated proportionally to items by value. |
| Landed Cost | Item landed cost = purchase price + allocated transport cost |
| Audit | All PO/Transport create/update/cancel actions logged with user + timestamp + changes |

### Advance Payment (Always against PI)

Client advance is always against a specific PI. Currency and conversion_rate are **auto-inherited from the linked PI** — you don't send them.

**Create Advance Payment:**
```
POST /api/finance/advance-payments
```
```json
{
  "proforma_invoice": "pi-uuid",
  "client_name": "ABC International",
  "amount": 100000.00,
  "payment_date": "2026-05-25",
  "payment_mode": "TT",
  "reference_number": "TT-REF-001",
  "remarks": "30% advance as per PI terms"
}
```

**Auto-populated from PI:**
- `currency` — inherited from PI's currency
- `conversion_rate` — inherited from PI's conversion_rate

**Auto-calculated:**
- `amount_inr` = amount × conversion_rate (or just amount if INR)
- `remaining` = amount - amount_used
- `advance_number` = `ADV/2026/0001` (auto)

**Response:**
```json
{
  "id": "uuid",
  "advance_number": "ADV/2026/0001",
  "proforma_invoice": "pi-uuid",
  "pi_number": "EEL/IND/PI/2026/0001",
  "client_name": "ABC International",
  "amount": 100000.00,
  "currency": "USD",
  "conversion_rate": 84.5000,
  "amount_inr": 8450000.00,
  "amount_used": 0,
  "remaining": 100000.00,
  "payment_date": "2026-05-25",
  "payment_mode": "TT",
  "status": "ACTIVE"
}
```

**List Advance Payments:**
```
GET /api/finance/advance-payments
GET /api/finance/advance-payments?status=ACTIVE&proforma_invoice=uuid&currency=USD
```

**Adjust (Use) Advance:**
```
POST /api/finance/advance-payments/{id}/adjust
```
```json
{
  "amount": 50000.00
}
```
Deducts from remaining. When remaining = 0, status = `FULLY_USED`.

**Note:** Advance payments cannot be deleted.

## Key Business Rules

| Rule | Description |
|------|-------------|
| **PI = Items Only** | PI has no GST, no discount. Only `grand_total` = sum of items. |
| **PI Bill = GST + Discount** | PI Bill is where GST and discount are applied. Generated after PI acceptance. |
| **Advance = Always PI** | Advance payments are always against a PI. Currency/conversion_rate auto-inherited from PI. |
| **Multi-Currency** | All amounts in original currency + immutable conversion_rate. P&L always in INR. |
| **PO Number** | `EEL/IND/<VENDOR_PREFIX>/<NUMBER>` starting from 100 |
| **PI Number** | `EEL/IND/PI/{YEAR}/{NUMBER}` starting from 0001 |
| **PI Bill Number** | `PIB/{YEAR}/{NUMBER}` starting from 0001 |
| **Advance Number** | `ADV/{YEAR}/{NUMBER}` starting from 0001 |
| **Revision** | First edit appends "R" to number. `revision_number` increments. |
| **Edit Lock** | 30-minute timeout. Must lock before editing PO/PI. |
| **Password** | Cancel PO, Cancel PI Bill, Record PO Payment, Mark PI Bill Paid — all require `confirm_password`. |
| **GST** | CGST + SGST (intra-state) OR IGST (inter-state). Applied on items_total/subtotal. |
| **Stock** | Auto-updated when PO item marked as purchased. Reversed on PO cancel. |
| **Transport** | Multiple shipments per PO/PI. Cost allocated proportionally by item value. |
| **Deletion** | Requisitions, Assignments, Quotations, POs, PI Bills, Advance Payments cannot be deleted. |

    |

                  |
                  └──→ CANCELLED
```

```

    |
    └──→ CANCELLED

    |
